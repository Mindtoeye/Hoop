Each hoop should come with its own help file based on this template file. Please use this to generate standardized documentation by filling in at least the lists shown below. When creating a new template, please cover the following topics:

  1. What data types does the hoop take, if any
  2. For each data type the hoop processes, what are the assumptions and requirements? For example a particular hoop might only take a list of String KVs
  4. What can a user expect the transformation to be like? What will the resulting KV model be?
  5. Are there any combinations of KV inputs that are preferable over others? If so, what are they?
